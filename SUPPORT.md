# Support

## 1. Overview

This document outlines how to get help, report issues, and interact with projects within the Novum Systems organization.

All communication should follow the structured workflows defined for each repository.

---

## 2. Reporting Issues

If you encounter a problem or bug:

- Use the **Bug Report** issue template
- Provide clear steps to reproduce the issue
- Include relevant environment details

Incomplete reports may not be addressed.

---

## 3. Feature Requests

If you would like to suggest a feature:

- Use the **Feature Request** template
- Clearly describe the problem and proposed solution
- Explain the value or impact of the feature

---

## 4. Questions and Discussions

For general questions:

- Use the **Question** issue template
- Keep questions specific and relevant to the project

---

## 5. Security Issues

⚠️ Do NOT report security vulnerabilities publicly.

Instead:
- Use GitHub Security Advisories where available
- Or contact the repository owner directly

---

## 6. Contribution Guidance

If you intend to contribute:

- Review the `CONTRIBUTING.md` file
- Follow the defined workflow and standards
- Ensure your work aligns with project goals

---

## 7. Response Expectations

- Not all requests or issues will be acted upon
- Priority is determined based on project goals and available resources
- Responses may vary depending on workload

---

## 8. Scope of Support

Support is limited to:

- Issues directly related to the repository
- Valid bugs or feature requests
- Relevant technical questions

Support does NOT include:

- General programming help unrelated to the project
- Custom implementation support
- Private consulting or development requests

---

## 9. Professional Conduct

All interactions are expected to follow the Code of Conduct.

Unproductive or inappropriate interactions may be ignored or removed.
