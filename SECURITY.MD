# Security Policy

## 1. Overview

The Novum Systems organization takes security seriously.  
This document outlines how to responsibly report vulnerabilities and how they will be handled.

---

## 2. Scope

This policy applies to all repositories within the Novum Systems organization unless stated otherwise.

If a repository includes its own SECURITY.md file, that file takes precedence.

---

## 3. Reporting a Vulnerability

If you discover a security vulnerability, **do not disclose it publicly**.

Instead, report it through one of the following:

- GitHub Security Advisories (preferred)
- Direct contact with the repository owner (if necessary)

### When reporting, please include:

- A clear description of the vulnerability
- Steps to reproduce the issue
- Potential impact
- Any suggested mitigations (if available)

Incomplete reports may delay investigation.

---

## 4. Responsible Disclosure

To protect users and systems:

- Do NOT open public issues for security vulnerabilities
- Do NOT share exploits or sensitive details publicly
- Allow time for the issue to be investigated and resolved before disclosure

Public disclosure without coordination may result in the report being disregarded.

---

## 5. Response Process

While response times may vary, the general process is:

1. Report is received and reviewed
2. Vulnerability is validated
3. Fix is developed and tested
4. Patch is released
5. Disclosure may occur after resolution

---

## 6. Response Expectations

- Not all reports will result in immediate action
- Priority is based on severity and impact
- There is no guaranteed response time

---

## 7. Limitations

- This is an independent development environment
- Formal bug bounty programs are not offered
- No compensation is guaranteed for vulnerability reports

---

## 8. Security Philosophy

Novum Systems follows a practical approach to security:

- Minimise attack surface
- Maintain clear and auditable systems
- Prioritise secure design over reactive fixes

---

## 9. Contact

For security-related inquiries, use the reporting methods above.
